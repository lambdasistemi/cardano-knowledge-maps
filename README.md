# Cardano Governance Graph

An interactive knowledge graph of Cardano's on-chain governance system (CIP-1694, Conway era).

**Live:** https://lambdasistemi.github.io/cardano-governance-graph/

## What is this?

A browsable graph that maps the relationships between all the entities, processes, rules, and parameters that make up Cardano governance. Click any node to see its description and links to source documentation. Click edges to understand why relationships exist.

The graph covers:

- **Actors** — Ada holders, DReps, SPOs, Constitutional Committee, Cardano Foundation, Intersect
- **Governance action types** — all 7 types defined in CIP-1694 (No-Confidence, Committee Update, New Constitution, Hard Fork, Parameter Changes, Treasury Withdrawal, Info)
- **Processes** — voting, ratification, enactment, action chaining, bootstrap phase
- **Parameters** — Network, Economic, Technical, and Governance parameter groups, security-relevant parameters, Plutus cost models
- **Mechanisms** — voting thresholds (DRep, SPO, CC), deposit mechanism, hot/cold keys, pre-defined voting options (Abstain, No Confidence)
- **Framework** — CIP-1694, Conway era, Cardano Constitution, Guardrails Script, metadata standards
- **Tools** — GovTool, SanchoNet

41 nodes, 66 edges, each with descriptions and links to official documentation.

## How to use

- **Click** a node to re-center the graph on it and see its description in the sidebar
- **Hover** an edge to see why the relationship exists
- **Depth buttons** (1, 2, 3, All) control how many hops from the selected node are shown
- **Fit View** re-fits the viewport to all visible nodes
- Links in the sidebar open official Cardano documentation (CIPs, docs.cardano.org, developers.cardano.org, Cardano Foundation)

## Tech stack

- **PureScript** + **Halogen** — typed functional UI
- **Cytoscape.js** + **fCoSE** — graph visualization with force-directed layout
- **Nix flake** — reproducible development environment
- **Spago** — PureScript package manager and bundler
- **GitHub Pages** — deployment via GitHub Actions
- **Surge.sh** — PR preview deploys

## Data

The graph data lives in `data/graph.json`. Node descriptions and links are sourced from:

- [CIP-1694](https://cips.cardano.org/cip/CIP-1694) — the governance specification
- [docs.cardano.org](https://docs.cardano.org/about-cardano/governance-overview) — Cardano documentation
- [developers.cardano.org](https://developers.cardano.org/docs/governance/cardano-governance/governance-model/) — developer documentation
- [Cardano Foundation governance materials](https://cardanofoundation.org/governance) — educational content

## Disclaimer

All graph data — node descriptions, edge descriptions, relationship mappings, and link selections — was generated by AI (Claude, Anthropic) based on publicly available Cardano governance documentation. While care was taken to source descriptions from official Cardano Foundation and Intersect materials, the content has not been formally reviewed or endorsed by any Cardano entity. Errors, omissions, or mischaracterizations may exist. Always verify governance details against the [official CIP-1694 specification](https://cips.cardano.org/cip/CIP-1694) and [Cardano documentation](https://docs.cardano.org) before acting on any information presented here.
